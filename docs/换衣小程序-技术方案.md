# 换衣小程序 — 技术方案与架构建议

> 面向电商推广场景的 AI 虚拟试衣小程序，一次开发，微信 + 抖音双端发布

---

## 一、方案概览

| 项目 | 选型 |
|------|------|
| 前端框架 | uni-app + Vue 3 |
| AI 能力 | 阿里云百炼 OutfitAnyone（aitryon） |
| 模特来源 | 预设模特图 + 用户上传 |
| 目标平台 | 微信小程序、抖音小程序 |

---

## 二、整体架构

```
┌─────────────────────────────────────────────────────────────────┐
│                        小程序端 (uni-app)                          │
│  ┌──────────────┐  ┌──────────────┐  ┌──────────────────────────┐ │
│  │ 选择/上传模特  │  │ 选择/上传服饰  │  │ 结果展示 / 下载 / 分享     │ │
│  └──────┬───────┘  └──────┬───────┘  └────────────▲─────────────┘ │
│         │                 │                        │              │
│         └─────────────────┼────────────────────────┘              │
│                           │ 上传图片、发起试衣、轮询结果              │
└───────────────────────────┼───────────────────────────────────────┘
                            │ HTTPS
                            ▼
┌─────────────────────────────────────────────────────────────────┐
│                      自建后端服务 (必需)                           │
│  ┌─────────────────────────────────────────────────────────────┐ │
│  │ 1. 图片上传：接收小程序上传 → 转存 OSS/临时存储 → 生成公网 URL   │ │
│  │ 2. AI 调用：使用公网 URL 调用百炼 API，创建试衣任务              │ │
│  │ 3. 结果轮询：按 task_id 查询任务状态，返回结果图 URL            │ │
│  └─────────────────────────────────────────────────────────────┘ │
└───────────────────────────┬───────────────────────────────────────┘
                            │ HTTPS (API Key 仅后端持有)
                            ▼
┌─────────────────────────────────────────────────────────────────┐
│              阿里云百炼 OutfitAnyone API                          │
│  输入：模特图 URL + 服饰图 URL → 输出：试衣效果图 URL               │
└─────────────────────────────────────────────────────────────────┘
```

**为何必须自建后端：** 百炼 API 要求输入为**公网可访问的 HTTP/HTTPS URL**，且 API Key 不能暴露在小程序端，因此必须由后端负责图片上传与 API 调用。

## 三、后端架构（Python FastAPI）

### 3.0 技术选型

| 组件 | 选型 |
|------|------|
| 框架 | FastAPI |
| 图片存储 | 阿里云百炼临时存储（model 绑定 aitryon）或 公网 OSS |
| AI 调用 | httpx / requests 调用百炼 API |
| 部署 | 推荐 Docker + 云服务器（如阿里云 ECS） |

### 3.0.1 目录结构

```
backend/
├── app/
│   ├── __init__.py
│   ├── main.py              # FastAPI 应用入口
│   ├── config.py            # 配置（API Key、上传限制等）
│   ├── routers/
│   │   ├── upload.py        # 图片上传
│   │   └── tryon.py         # 试衣创建、结果查询
│   ├── services/
│   │   ├── oss_upload.py    # 百炼临时存储上传
│   │   └── bailian.py       # 百炼 API 调用封装
│   └── schemas/
│       └── models.py        # Pydantic 请求/响应模型
├── requirements.txt
├── Dockerfile
└── .env.example             # 环境变量模板
```

### 3.0.2 图片上传流程

小程序 `multipart/form-data` 上传 → FastAPI `UploadFile` 接收 → 临时落盘 → 调用百炼 `getPolicy`（model=aitryon）→ 上传至百炼临时 OSS → 返回 `oss://...` URL。调用试衣 API 时需在 Header 添加 `X-DashScope-OssResourceResolve: enable`。

### 3.0.3 依赖

```
fastapi
uvicorn[standard]
python-multipart
httpx
python-dotenv
```

### 3.0.4 环境变量

| 变量 | 说明 |
|------|------|
| `DASHSCOPE_API_KEY` | 阿里云百炼 API Key（中国内地北京地域） |

## 四、核心模块设计

### 4.1 模特来源

| 方式 | 实现 |
|------|------|
| 预设模特 | 将若干符合 API 要求的全身正面照打包到小程序资源，或存于 CDN，用户选择后直接使用其公网 URL |
| 用户上传 | 用户通过 `uni.chooseImage` 选择相册/拍照 → 上传至后端 → 后端转存并返回公网 URL |

预设模特建议至少 3–5 套（男/女、不同体型），便于快速出图；上传模特需做基础校验（尺寸、格式、大小）。

### 4.2 服饰来源

| 方式 | 实现 |
|------|------|
| 用户上传 | 同模特，选择图片 → 上传 → 获取公网 URL。需提示：平铺、无褶皱、背景干净 |

### 4.3 试衣流程

1. 用户选择/上传模特图、服饰图
2. 前端调用后端「创建试衣任务」接口，传入两张图片的公网 URL
3. 后端调用百炼 API 创建任务，返回 `task_id`
4. 前端按间隔（如 2–3 秒）轮询后端「查询任务结果」接口
5. 任务完成后展示结果图，支持保存、分享

## 五、后端接口设计

| 接口 | 方法 | 说明 |
|------|------|------|
| `/api/upload/image` | POST | 上传图片，返回公网 URL（或 oss:// URL，后端调用 API 时携带 `X-DashScope-OssResourceResolve: enable`） |
| `/api/tryon/create` | POST | 创建试衣任务，入参：person_image_url、garment_image_url，返回 task_id |
| `/api/tryon/result` | GET | 查询任务结果，入参：task_id，返回 status、result_image_url |

可选：增加用户登录、试衣记录、限流等，便于后续运营与计费控制。

## 六、双端兼容要点（uni-app）

| 维度 | 说明 |
|------|------|
| 图片选择 | 使用 `uni.chooseImage` 或 `uni.chooseMedia`，框架自动兼容微信/抖音 |
| 网络请求 | 使用 `uni.request` 替代原生 `wx.request` / `tt.request` |
| 分享 | 微信用 `uni.share`，抖音需按平台文档做挂载/分享适配 |
| 包体积 | 微信主包 ≤ 2MB，抖音主包 ≤ 4MB；预设模特建议放 CDN 或分包，避免主包过大 |

---

## 七、成本与限流（百炼 aitryon）

| 项目 | 数值 |
|------|------|
| 单价 | 约 0.20 元/张 |
| 限流 | 10 RPS（主账号与子账号共享） |
| 免费额度 | 约 400 张（以官方文档为准） |

建议：在前端增加排队/限流提示，后端统一封装调用，便于后续切换模型或接入其他 AI 供应商。

| 阶段 | 内容 | 预估 |
|------|------|------|
| 1 | 搭建 uni-app 空项目，配置微信、抖音编译目标 | 1–2 天 |

## 十、风险与应对

| 风险 | 应对 |
|------|------|
| 用户上传图片不符合 API 要求 | 前端强提示 + 后端基础校验，失败时返回清晰错误文案 |
| 抖音/微信审核差异 | 提前查阅各平台小程序规范，避免敏感词和违规能力 |
| 百炼 API 变更 | 抽象 AI 调用层，便于后续更换或并行接入其他模型 |

## 十一、附录：百炼 API 快速参考

- 创建任务：`POST https://dashscope.aliyuncs.com/api/v1/services/aigc/image2image/image-synthesis`
- Header：`X-DashScope-Async: enable`，`Authorization: Bearer <API Key>`
- Body：`model: "aitryon"`，`person_image_url`，`garment_image_url`
- 查询结果：`GET https://dashscope.aliyuncs.com/api/v1/tasks/{task_id}`

详细参数见 [阿里云帮助文档](https://help.aliyun.com/zh/model-studio/outfitanyone-api)。
